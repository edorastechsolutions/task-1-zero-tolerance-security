# Week 4 – Full Web Application Penetration Testing

## Objective
The objective of this task is to simulate a real-world web application penetration testing process by identifying, exploiting, and analyzing vulnerabilities using both manual and automated security testing techniques.

## Application Tested
DVWA (Damn Vulnerable Web Application)

## Tools Used
- Burp Suite
- OWASP ZAP
- Nikto
- Browser Developer Tools

## Methodology
1. Reconnaissance
2. SQL Injection Testing
3. Cross-Site Scripting Testing
4. Session Hijacking
5. IDOR Testing
6. Request Manipulation
7. Automated Scanning
8. Server Misconfiguration Testing

## Key Findings
- SQL Injection Vulnerability
- Cross-Site Scripting (XSS)
- Broken Access Control (IDOR)
- Session Hijacking Risk

## Outcome
Multiple vulnerabilities were identified showing improper input validation, weak session management, and insecure access control mechanisms.
